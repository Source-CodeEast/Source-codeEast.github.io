    使用springInitalizer快速创建spring Boot项目

    a.安装Spring Asistant插件（针对于社区版本的idea）与创建项目：

    1.由于运营商屏蔽Idea插件安装商城，因此需要通过官网https://plugins.jetbrains.com/plugin/10229-spring-assistant?search=spring%20As来下载spring Asistant插件

    2.选择完版本号后下载，将文件解压

    3.导入idea中file -> setting ->plugin ->Intall Plugin from Disk ;之后指定jar包导入即可

    001.png

    4.此时便可new project来选择springInitalizer的快捷方式创建springBoot项目；若出现无法正常使用，可以在来到plugins ->Installed目录下选择Spring Asistant插件选择更新便可；

    b.使用Spring Initializer快速创建项目：

    1.IDE都支持使用Spring的项目创建向导快速创建一个Spring Boot项目；
    选择我们需要的模块；向导会联网创建Spring Boot项目； 默认生成的Spring Boot项目；
    主程序已经生成好了，我们只需要我们自己的逻辑 resources文件夹中目录结构
    static：保存所有的静态资源； js css images；
    templates：保存所有的模板页面；（Spring Boot默认jar包使用嵌入式的Tomcat，默认不支持 JSP页 面）；可以使用模板引擎（freemarker、thymeleaf）；
    application.properties：Spring Boot应用的配置文件；可以修改一些默认设置；

    c.配置文件

    1.springBoot使用全局配置文件，配置文件名是固定的：

    • application.properties application.yml

      2.配置文件的作用：修改SpringBoot自动配置的默认值；SpringBoot在底层都给我们自动配置好；
    3.yml与xml区别为：
    • yml：以数据为中心，比json，xml等更适合做配置文件
    • 类似于键值对（key: value）
    • 以空格的缩进来控制层级关系；只要是左对齐的一列数据，都是同一个层级的
    • 在： 之后需要一个空格（这点对于yml非常重要）
    4.配置文件的注入（配置文件与javaBean中数据的传输）如果出现问题就更新xml后右键xml 重新加载：

    • 将配置文件中配置的每一个属性的值，映射到这个组件中

    • @ConfigurationProperties：告诉SpringBoot将本类中的所有属性和配置文件中相关的配置进行绑定；

    • prefix = “person”：配置文件中哪个下面的所有属性进行一一映射 ；使用方法为@ConfigurationProperties(prefix = “person”)

    • 加上一个容器@Component；才能容器提供的@ConfigurationProperties功能；
